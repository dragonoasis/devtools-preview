设备信息查看器 - 浏览器、屏幕和硬件详情

查看详细的设备和浏览器信息，包括用户代理、屏幕分辨率、GPU、CPU 核心数、内存、网络连接类型等——全部实时检测。

Device Information

什么是设备信息查看器？

设备信息查看器是一个客户端工具，通过 Navigator API、Screen API、WebGL 和 Network Information API 等标准 Web API 读取您的浏览器和设备属性。它以结构化的方式展示您当前的浏览器信息、屏幕尺寸、系统硬件和 GPU 能力。所有数据均在浏览器本地采集，不会发送到任何服务器。

如何使用设备信息查看器

  1. 打开工具——您的设备信息会自动检测并显示。
  2. 浏览四个板块：浏览器、屏幕、系统和 GPU。
  3. 点击任意数值旁边的复制按钮，将其复制到剪贴板。
  4. 点击刷新按钮重新读取所有数值（在调整窗口大小或改变屏幕方向后很有用）。
  5. 使用显示的 User Agent 字符串和屏幕信息来调试响应式布局或浏览器特定问题。

常见使用场景

  • Bug 报告和支持工单 — 快速收集准确的浏览器版本、操作系统平台、屏幕分辨率和 GPU 信息，附在 Bug 报告或支持工单中以便准确复现。
  • 响应式设计测试 — 查看实际屏幕尺寸、像素比率和方向，验证网站在当前设备上的渲染效果。
  • 性能诊断 — 查看 CPU 核心数、设备内存、GPU 渲染器和网络连接类型，了解当前设备的性能特征。
  • User Agent 验证 — 检查完整的 User Agent 字符串和解析后的浏览器名称/版本，调试 UA 嗅探逻辑或验证功能检测结果。

FAQ

数据会被发送到服务器吗？
不会。所有信息均通过浏览器原生 JavaScript API 读取，完全保留在您的浏览器中，不会传输到任何外部服务器。
为什么有些值显示 N/A？
Device Memory 和 Network Information 等 API 并非所有浏览器都支持（特别是 Safari 和 Firefox）。当 API 不可用时，工具会显示 N/A。
GPU 信息准确吗？
GPU 信息通过 WebGL 的 WEBGL_debug_renderer_info 扩展读取。大多数桌面浏览器会暴露真实的 GPU 名称，但一些注重隐私的浏览器或配置可能会屏蔽或隐藏此数据。
调整浏览器窗口大小后屏幕尺寸会变化吗？
屏幕宽度和高度反映的是物理显示器尺寸，而非浏览器视口。无论窗口大小如何变化，它们保持不变。如果将窗口移动到不同 DPI 的显示器，像素比率可能会改变。
