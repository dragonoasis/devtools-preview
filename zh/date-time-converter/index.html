什么是日期时间转换器？

日期时间转换器可以将日期和时间从一个时区转换为另一个时区，并以多种标准格式显示结果。本工具使用浏览器内置的 Intl.DateTimeFormat API 进行精确的 IANA 时区转换，无需外部库。输入任意日期时间，选择源时区和目标时区，即可看到 ISO 8601、RFC 2822、Unix 时间戳（秒和毫秒）、本地格式和相对时间的转换结果。

如何使用日期时间转换器

  1. 在输入框中输入日期时间字符串（如 '2024-06-15T14:30:00'），或点击「现在」按钮使用当前时间。
  2. 选择输入日期时间所在的源时区。
  3. 选择要转换到的目标时区。
  4. 查看六种格式的转换结果：ISO 8601、RFC 2822、Unix 秒级时间戳、Unix 毫秒级时间戳、本地格式和相对时间。
  5. 点击每行结果旁边的复制按钮，将其复制到剪贴板。

常见使用场景

  • 跨时区会议安排 — 在不同时区的办公室之间转换会议时间，避免安排国际电话或部署时产生混淆。
  • API 与日志时间戳转换 — 将服务器日志或 API 响应中的 UTC 时间戳转换为本地时区，加速调试效率。
  • 定时任务验证 — 验证以 UTC 配置的 Cron 定时任务是否会在预期的本地时间触发。
  • ISO 8601 和 RFC 2822 格式化 — 生成格式正确的日期时间字符串，用于 HTTP 头、邮件 Date 字段或 API 请求体。

FAQ

支持哪些时区？
本工具包含 13 个最常用的 IANA 时区，覆盖 UTC、美洲、欧洲、亚洲、澳洲和太平洋地区。转换基于浏览器原生 Intl API，自动处理夏令时规则。
是否支持夏令时？
是的。由于使用了浏览器的 Intl.DateTimeFormat 和 IANA 时区标识符，夏令时转换会根据输入日期自动应用。
可以输入哪些日期时间格式？
支持 JavaScript Date 构造函数能解析的所有格式，包括 ISO 8601（2024-06-15T14:30:00）、纯日期字符串（2024-06-15）以及常见格式如 'Jun 15 2024 14:30'。输入会按照所选源时区解释。
为什么切换目标时区时 Unix 时间戳不变？
Unix 时间戳表示一个绝对时间点（从 UTC 的 Unix 纪元起经过的秒数），与显示时区无关——只有人类可读的格式会随时区变化。
