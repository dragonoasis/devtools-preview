URL Slug 生成器

免费在线 URL Slug 生成器。将任意文本转换为 SEO 友好的 URL 安全别名，支持自定义分隔符、大小写选项和长度限制。

什么是 URL Slug 生成器？

URL Slug 生成器通过去除特殊字符、用连字符或其他分隔符替换空格以及规范化 Unicode 字符，将可读文本转换为 URL 友好的字符串。Slug 是 URL 中以易读形式标识页面的部分，例如 example.com/blog/my-blog-post 中的 'my-blog-post'。精心制作的 slug 可以提高网页的 SEO 表现、可读性和可分享性。

如何使用 URL Slug 生成器

  1. 在输入文本区域中输入或粘贴要转换的文本。
  2. 选择您偏好的分隔符：连字符（-）、下划线（_）或点（.）。
  3. 根据需要开启或关闭小写选项。
  4. 可选择设置最大长度以保持 slug 简洁。
  5. 使用复制按钮复制生成的 slug。

使用场景

  • 博客文章 URL — 将文章标题转换为干净、SEO 友好的 URL slug，使搜索引擎和用户易于理解内容描述。
  • 电商产品页面 — 从产品名称生成一致、可读的产品 URL，改善导航体验、SEO 排名和客户体验。
  • CMS 内容管理 — 在构建内容管理系统时从页面标题自动生成 URL slug，确保唯一且格式正确的永久链接。
  • 文件命名规范 — 清理用户提供的字符串以用作文件名或目录名，去除可能引起问题的特殊字符和空格。

FAQ

什么是好的 URL slug？
好的 slug 应该简短、描述性强、使用小写字母、用连字符分隔单词、避免特殊字符和停用词，并准确反映页面内容。为了 SEO 效果，建议将 slug 长度控制在 60 个字符以内。
URL slug 会影响 SEO 吗？
是的，URL slug 是一个次要但相关的 SEO 排名因素。搜索引擎使用 slug 中的关键词来理解页面内容。干净、描述性的 slug 还能提高搜索结果中的点击率，因为用户可以看到页面的内容。
此工具如何处理特殊字符和重音符号？
该工具使用 NFD 分解进行 Unicode 规范化，去除字符上的重音（例如 é 变为 e，ü 变为 u）。所有剩余的非字母数字字符被移除，空格被替换为所选的分隔符。
URL 中应该使用连字符还是下划线？
Google 建议在 URL 中使用连字符（-）作为单词分隔符，因为 Google 将连字符视为单词分隔符，而将下划线视为单词连接符。例如，'my-page' 被解读为两个单词，而 'my_page' 可能被视为一个单词。
